## The crux (Chris's data-flow concern)

- App player card writes ratings/notes → **Supabase `player_evals` immediately**
  (durable). ✅ automatic.
- `player_evals` → **Sheet 2.0** happens **only** when `sync_player_evals.py --send`
  is run. **There was NO scheduler** (verified — no workflows/cron in either repo).
  So unless someone runs it, app evals sit unsynced in Supabase, NOT in the sheet.
- The nightly workflow is the fix. Safe unattended: dry-run-validates then aborts on
  error; append-only / only-if-changed (never deletes); idempotent by op_id;
  per-op quarantine; stamps `synced_at` only on server-confirmed rows.
- **Deploys don't touch data.** Frontend deploy = new JS only; DB rows untouched;
  game tags (`marks`) not touched by any change here; all writes append-only;
  Vercel deploys are session-safe. Recovery: Supabase Pro daily backups +
  Google Sheet Version History.
